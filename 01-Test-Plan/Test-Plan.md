# 🧪 Test Plan – Conduit (RealWorld App)

## 📌 Objective

The objective of this test plan is to define the testing strategy, scope, and approach used to validate the Conduit (RealWorld) web application.

This project aims to ensure the quality, functionality, and reliability of the application through structured manual testing.

---

## 🎯 Scope

The testing scope includes the following modules:

- Authentication
  - Login
  - Registration
- Articles
  - New Article
- Home Page
- User Settings

---

## 🧠 Testing Approach

The testing approach is based on **manual testing**, focusing on:

- Functional testing  
- Negative testing  
- Input validation  
- User interaction and navigation  
- System behavior under different scenarios  

Test cases were designed to cover both **valid and invalid inputs**, ensuring comprehensive validation of application behavior.

---

## 🧪 Test Design Techniques

The following techniques were applied:

- Equivalence Partitioning  
- Boundary Value Analysis  
- Negative Testing  
- Exploratory Testing  

---

## 📋 Test Suites

Two main test suites were created:

### 🔹 Smoke Test Suite
A subset of critical test cases designed to verify that the main functionalities of the application are working correctly.

### 🔹 Regression Test Suite
A complete set of test cases used to validate the stability of the application after changes or updates.

---

## 📊 Test Execution

Test cases were executed manually and documented in:

- Smoke Test Execution Report  
- Regression Test Execution Report  

Each report includes:

- Execution results (Pass / Fail)  
- Identified issues  
- Observations  
- Overall assessment  

---

## 🐞 Bug Reporting

Bugs were identified and documented during test execution.

Each bug report includes:

- Unique ID  
- Steps to reproduce  
- Expected vs actual result  
- Severity  
- Description  

---

## ⚙️ Test Environment

- Browser: Google Chrome  
- Operating System: Windows  
- Application: Conduit (RealWorld App)  

---

## ✅ Entry Criteria

- Application is accessible  
- Test environment is ready  
- Test cases are prepared  

---

## 🚪 Exit Criteria

- All planned test cases are executed  
- Critical defects are identified and documented  
- Test execution results are recorded  

---

## 📦 Test Deliverables

- Test Plan  
- Test Cases  
- Smoke Test Suite  
- Regression Test Suite  
- Test Execution Reports  
- Bug Reports  

---

## ⚠️ Risks

- Limited test data  
- Possible environment instability  
- Lack of backend access for deeper validation  

---

## 📈 Conclusion

The testing process provided a structured evaluation of the application, identifying multiple defects and validating key functionalities.

The results highlight areas for improvement, particularly in validation and user input handling, contributing to overall software quality.
